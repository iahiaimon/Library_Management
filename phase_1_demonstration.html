üéØ Phase 1 Overview & Project Structure

What is Phase 1?
Phase 1 introduces Django fundamentals through a simple book management system. You'll learn how the frontend (templates) communicates with the backend (views, models, and database) to create a complete web application.

How It Works: The Complete Flow

User Request ‚Üí URL Router ‚Üí View Function ‚Üí Model Query ‚Üí Template Rendering ‚Üí HTML Response to Browser Example: 1. User clicks "View All Books" 2. Browser sends request to /books/ 3. üîß Backend Components Working Together:

  • admin.py - Django admin registration for data management
‚ö†Ô∏è Key Concept: Templates are NOT just static HTML. They're dynamic documents that receive data from views and render it in real-time. Each template corresponds to a specific view function.

üóÑÔ∏è Backend: Database Models (models.py)

What Models Do: Models define the structure of your database tables. Django converts Python classes into SQL tables automatically.

The Book Model - Core of Phase 1

myapp/models.py - Book Model Definition
from django.db import models

class Book(models.Model):
    # Book title - stored as text in database
    title = models.CharField(
        max_length=200,
        help_text=\"Enter book title\"
    )
    
    # Author name
    author = models.CharField(
        max_length=100,
        help_text=\"Enter author name\"
    )
    
    # ISBN - unique identifier
    isbn = models.CharField(
        max_length=13,
        unique=True,
        help_text=\"Enter 13-digit ISBN\"
    )
    
    # How many copies in inventory
    quantity = models.IntegerField(
        default=1,
        help_text=\"Number of copies available\"
    )
    
    # When was it published
    publication_date = models.DateField(
        help_text=\"Select publication date\"
    )
    
    # Longer book description
    description = models.TextField(
        blank=True,
        help_text=\"Optional: Book summary\"
    )
    
    # When this record was created in database
    created_at = models.DateTimeField(auto_now_add=True)
    
    # Last time this record was updated
    updated_at = models.DateTimeField(auto_now=True)
    
    # How book appears in admin and lists
    def __str__(self):
        return f\"{self.title} by {self.author}\"
    
    # Metadata for model
    class Meta:
        ordering = ['-created_at']  # Newest first
        verbose_name = \"Book\"
        verbose_name_plural = \"Books\"
üìù Model Field Explanation:
  • CharField - Short text (max 200 chars)
  • IntegerField - Whole numbers (quantity)
  • DateField - Dates only (publication_date)
  • TextField - Long text (description)
  • DateTimeField - Date + time (created_at)
  • unique=True - No duplicate ISBNs allowed
  • auto_now_add=True - Automatically set when created
  • auto_now=True - Automatically update on save

üé¨ Backend: View Functions (views.py)

What Views Do: Views are Python functions that receive HTTP requests, query the database using models, and return HTML responses (using templates).

View 1: List All Books

myapp/views.py - book_list Function
from django.shortcuts import render
from .models import Book

# View: Display all books
def book_list(request):
    \"\"\" 
    Fetches all books from database and passes to template
    
    REQUEST FLOW:
    1. User clicks /books/
    2. Django routes to this function
    3. We query database: Book.objects.all()
    4. We render template with books data
    5. HTML sent back to browser
    \"\"\"
    
    # Query database for all books
    books = Book.objects.all()
    
    # Data to pass to template
    context = {
        'books': books,  # Available in template as {{ books }}
        'count': books.count()  # Available as {{ count }}
    }
    
    # Render template with data
    return render(request, \"myapp/book_list.html\", context)
üîç Step-by-Step Breakdown:
  1. Book.objects.all() - Queries database, returns all books
  2. context = {} - Dictionary of data for template
  3. render() - Combines template + data into HTML
  4. Template receives books and count variables

View 2: Create New Book

myapp/views.py - create_book Function
from django.shortcuts import render, redirect
from django.contrib import messages

# View: Add new book to database
def create_book(request):
    \"\"\"
    GET: Shows empty form
    POST: Saves new book to database
    \"\"\"
    
    if request.method == \"POST\":
        # Form submitted - extract data
        title = request.POST.get(\"title\")
        author = request.POST.get(\"author\")
        isbn = request.POST.get(\"isbn\")
        quantity = request.POST.get(\"quantity\")
        publication_date = request.POST.get(\"publication_date\")
        description = request.POST.get(\"description\", \"\")
        
        # Create new book record in database
        book = Book.objects.create(
            title=title,
            author=author,
            isbn=isbn,
            quantity=int(quantity),
            publication_date=publication_date,
            description=description
        )
        
        # Show success message
        messages.success(request, f\"Book '{title}' added successfully!\")
        
        # Redirect to book list
        return redirect(\"myapp:book_list\")
    
    # GET request - show empty form
    return render(request, \"myapp/book_form.html\", {
        'title': \"Add New Book\"
    })
üîÑ Request Processing Flow:
  • GET Request: User visits /add-book/ ‚Üí Shows empty form
  • POST Request: User submits form ‚Üí Saves to database
  • request.POST.get() - Extracts form data
  • Book.objects.create() - Inserts new row in database
  • redirect() - Takes user to book list after save

View 3: View Single Book

myapp/views.py - book_detail Function
from django.shortcuts import get_object_or_404

# View: Show details of one book
def book_detail(request, pk):
    \"\"\"
    Get specific book by ID (pk)
    If not found, show 404 error page
    \"\"\"
    
    # Find book with this ID, or show 404
    book = get_object_or_404(Book, pk=pk)
    
    context = {'book': book}
    
    return render(request, \"myapp/book_detail.html\", context)
üìñ Key Function:
  • pk parameter comes from URL (e.g., /book/1/ ‚Üí pk=1)
  • get_object_or_404() - Safely retrieves single book
  • If book doesn't exist, Django shows 404 error page

View 4: Edit Book

myapp/views.py - edit_book Function
# View: Update existing book
def edit_book(request, pk):
    \"\"\"
    GET: Show form filled with current data
    POST: Update database with new data
    \"\"\"
    
    book = get_object_or_404(Book, pk=pk)
    
    if request.method == \"POST\":
        # Update fields
        book.title = request.POST.get(\"title\")
        book.author = request.POST.get(\"author\")
        book.quantity = int(request.POST.get(\"quantity\"))
        book.description = request.POST.get(\"description\", \"\")
        
        # Save changes to database
        book.save()
        
        messages.success(request, \"Book updated successfully!\")
        return redirect(\"myapp:book_detail\", pk=book.pk)
    
    # GET - show form with current data
    context = {
        'book': book,
        'title': \"Edit Book\"
    }
    return render(request, \"myapp/book_form.html\", context)

üîó Backend: URL Routing (urls.py)

What URLs Do: URLs map web addresses to view functions. When you visit /books/, Django looks for matching URL pattern and calls the corresponding view.
myapp/urls.py - URL Patterns
from django.urls import path
from . import views

app_name = \"myapp\"  # URL namespace

urlpatterns = [
    # SHOW ALL BOOKS
    # URL: /books/
    # Function: views.book_list
    path(\"books/\", views.book_list, name=\"book_list\"),
    
    # SHOW SINGLE BOOK DETAILS
    # URL: /book/1/, /book/5/ etc.
    # pk parameter passes book ID to view
    path(\"book//\", views.book_detail, name=\"book_detail\"),
    
    # ADD NEW BOOK
    # URL: /add-book/
    path(\"add-book/\", views.create_book, name=\"create_book\"),
    
    # EDIT BOOK
    # URL: /book/1/edit/, /book/5/edit/ etc.
    path(\"book//edit/\", views.edit_book, name=\"edit_book\"),
    
    # DELETE BOOK
    # URL: /book/1/delete/, /book/5/delete/ etc.
    path(\"book//delete/\", views.delete_book, name=\"delete_book\"),
]
üîç URL Pattern Breakdown:
  • path() - Matches exact URL string
  • <int:pk> - Captures number from URL and passes to view
  • name= - Django references URL by name (not hardcoded string)
  • app_name - Namespace to avoid name conflicts
üí° URL Naming Convention: Use {% url 'myapp:book_detail' pk=book.id %} in templates instead of hardcoding URLs. If you change the URL pattern, templates still work!

myapp/admin.py - Register Book Model
from django.contrib import admin
from .models import Book

# Option 1: Simple registration
admin.site.register(Book)

@admin.register(Book)
class BookAdmin(admin.ModelAdmin):
    # Columns shown in list view
    list_display = [\"title\", \"author\", \"isbn\", 
                    \"quantity\", \"created_at\"]
    
    # Add filters on right side
    list_filter = [\"created_at\", \"author\"]
    
    # Add search box
    search_fields = [\"title\", \"author\", \"isbn\"]
    
    # Read-only fields (cannot edit)
    readonly_fields = [\"created_at\", \"updated_at\"]
    
    # Group fields in form
    fieldsets = (
        (\"Book Information\", {
            'fields': (\"title\", \"author\", \"isbn\")
        }),
        (\"Inventory\", {
            'fields': (\"quantity\", \"publication_date\")
        }),
        (\"Details\", {
            'fields': (\"description\", \"created_at\", \"updated_at\")
        }),
    )
üéØ Admin Features:
  • list_display - What columns show in list view
  • search_fields - Makes fields searchable
  • list_filter - Add filters for quick sorting
  • readonly_fields - Prevent editing of certain fields
  • fieldsets - Organize form into sections
